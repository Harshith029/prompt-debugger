# Pattern Library (anthropic pack)

Before/after prompt patterns, indexed by [`index.json`](index.json). Each pattern is a markdown file mapping a rubric dimension and technique(s) to a concrete transformation.

**Completeness definition (M1 FR-5):** the library is complete when every rubric dimension (R1–R10) — the analyzer's finding surface — is covered by at least one pattern with an authored body. Coverage over dimensions, not techniques, is the definition because a pattern exists to show the fix for a *finding*; techniques are the means. (T5, "give Claude a role", has no dedicated pattern because no rubric dimension maps to it.)

**Status:** the index lists ten patterns, one per rubric dimension R1–R10, each with an authored before/after body; all `status: draft` — promotion is deferred to M2, when the analyzer first binds to entry statuses (recorded in the [M1 milestone record](../../../../../docs/releases/M1.md)). Enforced by `tests/test_knowledge_integrity.py`: every indexed `file` exists (KN-6), index and on-disk bodies are bijective, every body carries the required sections and a real before/after example, each body agrees with its index entry (id, title, dimensions, techniques), and every rubric dimension is covered (KN-9).

**Authoring rules:** pattern bodies must respect the rewrite policy's transformation rules (`core/knowledge/packs/common/rewrite-policy.json`): a pattern may make intent more explicit, never less, and may only use facts the user actually holds — the After examples model stating *existing* intent, never fabricating context. Concretely, an **After may contain only information present in its Before**; information the user holds but the prompt lacks is written as an **angle-bracket slot** (`<what the user would supply>`) or an explicit question — never as invented dates, metrics, thresholds, audiences, technologies, or domain facts, and never by narrowing or broadening the request. (Literal XML data tags demonstrated by a pattern, e.g. `<build-log>`, are output structure, not slots; the body says so where they appear.) Enforced executably: the integrity suite rejects an After whose numeric or file-like literals do not appear in its Before. Body structure: title, id/dimension/techniques/status header, "When it applies", "Transformation", "Example" with **Before**/**After**, "Why it helps".
